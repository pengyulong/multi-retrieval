# 多路召回框架技术文档

## 一、背景与目标

在 RAG（检索增强生成）系统中，单一召回方式往往难以满足复杂场景的需求。不同召回算法各有优劣：

| 召回方式 | 优势 | 劣势 |
|---------|------|------|
| 向量召回 | 语义理解能力强 | 对关键词不敏感、计算开销大 |
| BM25 | 关键词匹配精准 | 无法捕捉语义相似性 |
| 分片召回 | 适合大规模数据 | 需要额外管理分片逻辑 |

**目标**：设计一个统一的多路召回框架，整合多种召回策略，通过融合算法输出高质量的候选结果。

---

## 二、整体架构

```
Query
  │
  ├──► [Vector Retriever] ──► Top-K vectors ──┐
  │                                          │
  ├──► [BM25 Retriever] ───► Top-K keywords ─┤
  │                                          │ Merge
  └──► [Shard Retriever] ──► Top-K shards ───┘
                                                      │
                                              ┌───────▼───────┐
                                              │  Score Fusion │
                                              │  (Weighted/RRF)│
                                              └───────┬───────┘
                                                      │
                                              ┌───────▼───────┐
                                              │  Final Top-K  │
                                              └───────────────┘
```

## 三、模块设计

### 3.1 向量召回模块 (VectorRetriever)

**职责**：基于语义向量相似度进行召回。

**核心组件**：
- `BaseVectorStore` - 向量存储抽象基类
- `MemoryVectorStore` - 内存向量存储（测试用）
- `FAISSVectorStore` - FAISS 向量索引

**核心流程**：
1. 文档通过 Embedder 编码为向量
2. 向量存储到向量数据库
3. 查询时，Query 编码为向量
4. 计算 Query 向量与文档向量的相似度
5. 返回 Top-K 最相似文档

**配置参数**：
```python
@dataclass
class VectorStoreConfig:
    backend: str = "memory"    # 存储后端
    dimension: int = 768       # 向量维度
    metric: str = "cosine"     # 相似度度量
    index_type: str = "flat"   # 索引类型
```

---

### 3.2 BM25 召回模块 (BM25Retriever)

**职责**：基于倒排索引的关键词稀疏检索。

**核心原理**：
BM25 公式：
```
Score(D, Q) = Σ IDF(qi) × (tf(qi, D) × (k1 + 1)) / (tf(qi, D) + k1 × (1 - b + b × |D|/avgdl))
```

其中：
- `tf(qi, D)` - 词项在文档中的频率
- `IDF(qi)` - 逆文档频率
- `k1`, `b` - 可调参数（默认 1.5, 0.75）

**核心流程**：
1. 分词（内置 jieba 中文支持）
2. 构建倒排索引
3. 查询时分词后查倒排表
4. 计算 BM25 分数
5. 返回 Top-K 文档

**配置参数**：
```python
@dataclass
class BM25Config:
    k1: float = 1.5   # 词频饱和参数
    b: float = 0.75   # 长度归一化参数
```

---

### 3.3 分片召回模块 (ShardRetriever)

**职责**：将大规模数据分散到多个分片，支持并行检索。

**分片策略**：
1. **Hash 分片** - 按 ID 哈希值取模分配
2. **Range 分片** - 按 ID 范围分配
3. **Round-Robin** - 轮询分配

**核心流程**：
1. 文档根据分片策略分配到对应分片
2. 查询时可指定分片或全部分片
3. 各分片并行/串行执行本地搜索
4. 汇总各分片结果

**配置参数**：
```python
@dataclass
class ShardConfig:
    num_shards: int = 4           # 分片数量
    shard_key: str = "id"         # 分片键
    routing_strategy: str = "hash" # 路由策略
```

---

### 3.4 统一召回器 (UnifiedRetriever)

**职责**：整合多路召回，执行分数融合。

#### 3.4.1 加权分数融合

1. 各路召回独立返回 Top-K 结果
2. 对各路分数做归一化（Min-Max / L2）
3. 按权重加权求和
4. 排序输出 Final Top-K

```
FinalScore(doc) = w_vector × norm_vector + w_bm25 × norm_bm25 + w_shard × norm_shard
```

#### 3.4.2 RRF 融合 (Reciprocal Rank Fusion)

基于排名的融合，无需调参：
```
RRF(doc) = Σ 1 / (k + rank(doc in retrieval_i))
```

其中 `k` 为常数（默认 60），`rank` 为文档在该路召回中的排名（从 1 开始）。

**融合对比**：

| 特性 | 加权融合 | RRF 融合 |
|------|---------|---------|
| 调参需求 | 需要权重 | 无需调参 |
| 分数含义 | 相对分数 | 排名分数 |
| 适用场景 | 权重可设置 | 多路效果相近 |

---

### 3.5 重排序模块 (Reranker)

**职责**：对候选结果进行精细化排序。

**支持类型**：
1. **SimilarityReranker** - 基于 Embedding 相似度
2. **CrossEncoderReranker** - 基于 Cross-Encoder 模型
3. **CohereReranker** - 基于 Cohere Rerank API

## 四、使用示例

```python
from retrieval import UnifiedRetriever, RetrievalConfig
from retrieval.vector_retriever import VectorRetriever
from retrieval.bm25_retriever import BM25Retriever

# 1. 准备文档
documents = [
    {"id": "1", "text": "Python编程语言教程", "category": "programming"},
    {"id": "2", "text": "机器学习入门", "category": "ai"},
    {"id": "3", "text": "深度学习实战", "category": "ai"},
]

# 2. 初始化召回器
vector_retriever = VectorRetriever(embedder=your_embedder)
bm25_retriever = BM25Retriever()

# 3. 构建索引
vector_retriever.index(documents)
bm25_retriever.index(documents)

# 4. 配置统一召回器
config = RetrievalConfig(
    enable_vector=True,
    enable_bm25=True,
    vector_weight=0.4,
    bm25_weight=0.6,
    use_rrf=False,
    final_top_k=10
)

unified = UnifiedRetriever(
    config=config,
    vector_retriever=vector_retriever,
    bm25_retriever=bm25_retriever
)

# 5. 执行召回
results = unified.retrieve("Python编程")

# 6. 查看各路召回详情
all_results = unified.retrieve("Python编程", return_all_scores=True)
```

## 五、配置参考

### 5.1 完整配置

```python
config = RetrievalConfig(
    # 启用控制
    enable_vector=True,
    enable_bm25=True,
    enable_shard=False,

    # Top-K 配置
    vector_top_k=20,
    bm25_top_k=20,
    shard_top_k=20,
    final_top_k=10,

    # 融合权重
    vector_weight=0.4,
    bm25_weight=0.4,
    shard_weight=0.2,

    # RRF 配置
    use_rrf=False,
    rrf_k=60,

    # 性能配置
    parallel=True,
    num_workers=4,

    # 归一化方法
    normalization="minmax"
)
```

### 5.2 场景化推荐

| 场景 | 配置建议 |
|------|---------|
| 语义为主 | vector_weight=0.7, bm25_weight=0.3 |
| 关键词为主 | vector_weight=0.3, bm25_weight=0.7 |
| 均衡召回 | use_rrf=True |
| 大规模数据 | enable_shard=True, parallel=True |
